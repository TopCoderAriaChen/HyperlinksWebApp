Exploring Hyperlinks

Overview

What are Hyperlinks?

Hyperlinks, usually called links, are the threads that weave together the World Wide Web, creating an intricate tapestry of interconnected resources. To explain what links are, we need to step back to the very basics of Web architecture.
At their core, links are directives embedded within web pages that guide users and browsers from one digital destination to another.
The inception of the web by Tim Berners-Lee in 1989 was marked by the introduction of three fundamental technologies:

  1. URL which serve as unique identifiers for resources
  2. HTTP as the protocol for transferring data across the web
  3. that structures content and enables the creation of hyperlinks

Together, these components constructed the bedrock of the internet's navigational capabilities.

In the web development context, hyperlinks are more than just pathways; they encapsulate user interaction, design paradigms, and the very essence of browsing. Although traditional hyperlinks initiate GET requests to retrieve documents, modern web applications often require a more dynamic approach. Here, AJAX (Asynchronous JavaScript and XML) comes into play, allowing for seamless interactions by making background HTTP requests without refreshing the entire page. This methodology enriches the user experience, fostering the development of responsive and adaptive web applications that react in real time to user input.

Note on Hyperlink Implementations:

  • Traditional hyperlinks are constrained to GET requests, such as when a user clicks a link to download a document. They cannot perform actions like POST or DELETE, which are essential in web applications for creating and removing content. To bridge this gap, AJAX enables asynchronous interactions, providing a means to achieve partial page refreshing. This functionality is key in creating sophisticated web experiences, such as single-page applications that maintain a fluid user interface while communicating with the server in the background. Modern implementations often utilize the Fetch API.
    • Asynchronous JavaScript and XML (Ajax, or AJAX) is a web development technique in which a web app fetches content from the server by making asynchronous HTTP requests, and uses the new content to update the relevant parts of the page without requiring a full page load. This can make the page more responsive, because only the parts that need to be updated are requested.
    • Ajax can be used to create single-page apps, in which the entire web app consists of a single document, which uses Ajax to update its content as needed.
    • Initially Ajax was implemented using the XMLHttpRequest interface, but the fetch() API is more suitable for modern web applications: it is more powerful, more flexible, and integrates better with fundamental web app technologies such as service workers. Modern web frameworks also provide abstractions for Ajax.

Types of Hyperlinks
Internal link

An internal link connects two pages within the same website. Websites, excluding those that consist of a single page, rely heavily on internal links to function.

Try it out:

Click Here


External link

An external link connects a page on your website to a page on another website. The Web is essentially a network of such links, which is vital for expanding the information beyond what is available on your own site.


Incoming links

These are links from other websites directing users to your website, essentially the reverse of an external link. Importantly, you're not required to reciprocate these links.

Anchors

Unlike typical links that connect separate webpages, Anchors tie two sections of one document together. When you follow a link pointing to an anchor, your browser jumps to another part of the current document instead of loading a new document. However, you make and use anchors the same way as other links.

Hyperlink Implementation in HTML

A basic link is created by wrapping the text or other content inside an <a> element and using the href attribute, also known as a Hypertext Reference, or target, that contains the web address.

html 
              
  <p>
    I'm creating a link to
    <a href="https://github.com/">the github homepage</a>.
  </p>

This gives us the following result:
I'm creating a link to the github homepage.
